# 강원 방언 페르소나 데이터셋 (ver3)

문진톡톡 파이프라인 검증용 더미 입력. 강원 방언 환자 발화와 표준어 답지(ground truth) 쌍을 담은 페르소나 데이터.

## 파일 구성

| 파일 | 내용 |
|---|---|
| `페르소나_ver3_정리.json` | 대사 본체 (방언 발화 + 표준어 답지) |
| `페르소나_ver3_meta.json` | 페르소나/시나리오 메타 (case_id로 join) |

두 파일은 **`case_id`로 1:1 매칭**된다. 총 20케이스 (페르소나 10명 × 초진/재진).

## case_id 규칙

```
ko_gw_p{번호}_{initial|followup}
```
- `ko_gw` : 한국어-강원
- `p1`~`p10` : 페르소나 번호
- `initial` : 초진 / `followup` : 재진

## 대사 파일 구조 (`페르소나_ver3_정리.json`)

```json
{
  "questions": {
    "초진": { "Q1": "...", "Q2": "...", "Q3": "...", "Q4": "..." },
    "재진": { "Q1": "...", "Q2": "...", "Q3": "...", "Q4": "..." }
  },
  "cases": [
    {
      "case_id": "ko_gw_p1_initial",
      "visit_type": "초진",
      "dialect_strength": "strong",
      "dialogue": {
        "Q1": { "answer": "방언 발화", "standard": "표준어 답지" },
        "Q2": { ... }, "Q3": { ... }, "Q4": { ... }
      }
    }
  ]
}
```

- **질문은 고정**이라 상단 `questions`에 초진/재진 한 번씩만 정리. 케이스에는 질문 텍스트 없이 `Q1~Q4` 키만 → 상단 `questions[visit_type][Qn]`으로 조회.
- `answer` : 환자/보호자의 강원 방언 발화 (STT가 받아쓸 실제 입력)
- `standard` : 같은 내용의 표준어 (정답지)
- 동반인이 있으면 한 줄에 화자 태그로 구분: `[환자] ... [보호자(딸)] ...`
  - 화자 종류: `[환자]`, `[배우자]`, `[요양보호사]`, `[보호자(딸)]`, `[보호자(아들)]`, `[보호자(아내)]`

## 메타 파일 구조 (`페르소나_ver3_meta.json`)

```json
[
  {
    "case_id": "ko_gw_p1_initial",
    "persona": "80s+ · 남 · 동반:배우자 · 감염급성/응급 · 어눌함",
    "scenario": "상황 설명 + 질환 + 어투 정보 ..."
  }
]
```

- `persona` 형식: `연령대 · 성별 · 동반:X · 증상카테고리/위험도 · 발화스타일`
- `scenario` : 배경·질환·지역 어투 등 상세 (방언 어투 지정 정보 포함)

## 필드 값 종류

| 필드 | 값 |
|---|---|
| `visit_type` | 초진 / 재진 |
| `dialect_strength` | strong / moderate |
| 연령대 | 60s / 70s / 80s+ |
| 동반 | 본인 단독 / 배우자 / 요양보호사 / 딸 / 아들 / 아내 |
| 증상카테고리 | 감염급성 / 간질성흉막 / 호흡곤란 / 만성기도 / 종양종괴 |
| 위험도 | 응급 / 주의 / 일반 |
| 발화스타일 | 어눌함 / 횡설수설 / 짧음 / 또렷함 / 다급함 |

## 규모

- 페르소나 10명 × (초진 + 재진)
- 케이스당 Q8 × (방언 + 답지) = 8줄 → **총 160줄**
